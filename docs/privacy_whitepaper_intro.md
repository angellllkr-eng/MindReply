# Privacy Whitepaper Intro

MindReply is built for private decision work. The system reads pressure, returns one synthesis, and recommends one action without turning sensitive material into a permanent visible record.

The privacy model is simple:

- Raw input is not stored in memory by default.
- Memory keeps derived preferences, not private wording.
- Receipts use hashes and metadata.
- High-risk signals pause movement before anything is sent.
- Access is controlled around the decision layer, not around a busy workspace.

This makes MindReply usable in executive, legal, clinical, financial, and operational contexts where trust matters more than speed.
